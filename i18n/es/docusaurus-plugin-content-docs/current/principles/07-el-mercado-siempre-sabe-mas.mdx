## Enunciado

Las señales que genera el uso real son datos de mayor calidad que cualquier hipótesis previa. El sistema debe estar diseñado para escucharlas y actuar sobre ellas. El mercado no miente, no especula y no tiene sesgos de confirmación.

## Por qué importa

El discovery más exhaustivo sigue siendo una hipótesis hasta que el mercado la valida o la invalida. Problem-Driven AI no trata de adivinar mejor, sino de crear un sistema que escuche mejor.

Las señales del mercado son datos de primera categoría: métricas de uso, feedback directo, patrones de abandono, solicitudes recurrentes. Ignorarlas en favor de las hipótesis internas es un acto de arrogancia metodológica.

## Implicaciones prácticas

- **Diseña para la observación**: cada entregable debe incluir mecanismos para capturar señales del mercado.
- **Prioriza datos de uso real** sobre opiniones internas o investigación teórica.
- **Establece ciclos de escucha**: no esperes a que el mercado grite. Diseña canales para escuchar sus susurros.

:::danger Anti-patrón: El Discovery Eterno

**Qué es.** Permanecer en la fase de investigación indefinidamente, refinando hipótesis sin exponerlas al mercado. La investigación se convierte en un refugio ante la incertidumbre del contacto con usuarios reales.

**Cómo detectarlo.** El equipo lleva múltiples ciclos en discovery sin producir un artefacto testeable. Cada hallazgo genera nuevas preguntas pero nunca una respuesta lo suficientemente clara para avanzar. La frase "necesitamos más datos" se usa para postergar decisiones, no para mejorarlas. No se han definido Exit Criteria para la fase de discovery.

**Cómo prevenirlo.** Define Exit Criteria para cada ciclo de discovery antes de que comience. Establece un número máximo de iteraciones por fase de discovery. Obliga al equipo a producir una hipótesis testeable tras cada ciclo, aunque sea imperfecta. El mercado refinará lo que la investigación no pueda.

:::
