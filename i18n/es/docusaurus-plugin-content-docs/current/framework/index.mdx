El Framework es la capa operativa de Problem-Driven AI. Mientras la metodología explica qué significa cada fase y por qué importa, el Framework indica exactamente cómo ejecutarla: procesos paso a paso, plantillas de artefactos, gate reviews, anti-patrones que evitar y estimaciones de esfuerzo. Es el puente entre comprender la metodología y aplicarla en un proyecto real.

<img src="/img/problem-driven-ai-phases.png" alt="Fases de Problem-Driven AI" style={{width: '100%', border: 'none', margin: '2rem 0'}} />

Cada card debajo enlaza a la guía operativa de una fase. Estas guías complementan el contenido conceptual que encontrarás en la sección de [Metodología](/es/fases). Donde la Metodología responde *¿qué es esta fase?* y *¿por qué existe?*, el Framework responde *¿cómo la ejecuto?*, *¿qué artefactos produzco?* y *¿cómo sé que he terminado?*. Ambas capas cubren las mismas cinco fases: empieza por Metodología para comprender, pasa al Framework para ejecutar.
