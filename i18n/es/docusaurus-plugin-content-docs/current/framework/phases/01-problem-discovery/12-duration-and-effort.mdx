# Duración y esfuerzo

El Problem Discovery no tiene una duración fija porque depende de la complejidad del problema, la accesibilidad de los usuarios y el nivel de conocimiento previo del equipo. Sin embargo, estas son las referencias típicas:

| Escenario | Duración | Qué incluye |
|---|---|---|
| **Problema simple, usuarios accesibles** | 1-2 semanas | 5 entrevistas + 1 sesión de síntesis + 1 ronda de validación. |
| **Problema de complejidad media** | 2-3 semanas | 5-8 entrevistas + observación contextual + 2 sesiones de síntesis + validación iterada. |
| **Problema complejo, múltiples stakeholders** | 3-5 semanas | 8-12 entrevistas con distintos perfiles + análisis cuantitativo + síntesis extendida + validación con múltiples grupos + verificación de restricciones. |

**Regla general:** si estás dedicando menos del 15% del tiempo total del proyecto al Problem Discovery, probablemente estás invirtiendo demasiado poco. Si estás dedicando más del 30%, probablemente estás sobreanalizando. El punto óptimo está entre el 15% y el 25% del tiempo total.
