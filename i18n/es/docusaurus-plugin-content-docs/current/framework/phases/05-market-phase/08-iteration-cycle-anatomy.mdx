# Anatomía del ciclo de iteración

El ciclo de Market Phase no es un proceso lineal que se ejecuta una vez. Es un sistema de **bucles anidados** que operan con cadencias distintas. Comprender esta anatomía es lo que permite al equipo operar la Fase 5 sin que consuma todo el tiempo disponible.

## Los tres bucles del ciclo

### Bucle interno: observación continua

**Cadencia:** Tiempo real → semanal.

| Actividad | Frecuencia | Automatización |
|---|---|---|
| Dashboards de KPIs actualizados | Tiempo real | 100% automático |
| Alertas por cruce de umbrales | Tiempo real | 100% automático |
| Revisión semanal de KPIs y comportamiento | Semanal (45-60 min) | Humano con datos automáticos |
| Entradas al Signal Log por capa automática | Continuo | 100% automático |

**Output:** Señales brutas capturadas y clasificadas preliminarmente. Alertas que requieren atención inmediata.

### Bucle medio: síntesis y actualización

**Cadencia:** Mensual.

| Actividad | Frecuencia | Automatización |
|---|---|---|
| Sesión de síntesis de feedback cualitativo | Mensual (2-3h) | Humano |
| Análisis de tickets y canales de soporte | Semanal-mensual | Parcial (síntesis con IA) |
| Actualización de Reglas y Story Files | Según necesidad | Humano (propuestas por IA en nivel 3+) |
| Context Update Record | Con cada actualización | Humano |

**Output:** Signal Log depurado con hipótesis de causa. Context Document actualizado. Context Update Record al día.

### Bucle externo: revisión estratégica

**Cadencia:** Trimestral.

| Actividad | Frecuencia | Automatización |
|---|---|---|
| Revisión estratégica con stakeholders | Trimestral (medio día) | Humano |
| Revisión de vigencia del Problem Statement | Trimestral | Humano |
| Análisis de señales externas y competitivas | Mensual-trimestral | Parcial |
| Redacción del Iteration Brief | Al cierre de cada ciclo | Borrador por IA (nivel 3+) |

**Output:** Iteration Brief del próximo ciclo. Decisión de regreso a fases anteriores si aplica.

## La cadena señal-causa-contexto

La cadena es el mecanismo central que transforma datos del mercado en actualizaciones del Context Document. Cada señal significativa debe recorrer la cadena completa antes de producir un cambio.

```
┌─────────────────────┐
│  Señal observada     │ ← Dato crudo del mercado
│  (comportamiento,    │
│   KPI, feedback)     │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│  Comportamiento      │ ← ¿Qué está haciendo el usuario?
│  detrás de la señal  │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│  Necesidad no        │ ← ¿Qué problema no resuelve
│  cubierta o problema │    la solución actual?
│  no resuelto         │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│  Elemento del        │ ← ¿Qué parte del Context
│  Context Document    │    Document no capturaba
│  afectado            │    esta realidad?
└─────────────────────┘
```

:::warning[Regla de oro]
Si el equipo no puede completar la cadena hasta el elemento del Context Document que debe actualizarse, **no tiene suficiente comprensión de la señal para actuar sobre ella**. Necesita más datos o más profundidad de análisis.
:::

## El flujo de actualización

No todas las actualizaciones del Context Document tienen el mismo proceso ni la misma urgencia.

| Urgencia | Tipo de cambio | Tiempo de respuesta | Aprobación |
|---|---|---|---|
| **Inmediata** | Regla técnica que afecta operación en producción | Mismo día | Context Engineer |
| **Normal** | Ajuste de Story Files, nueva Regla por patrón emergente | 1-3 días, próxima iteración | Context Engineer + Product Lead |
| **Estratégica** | Revisión de Solution Brief o Problem Statement | Semanas. Regreso formal a Fase 1 o 2. | Stakeholders completos |

## Señales de un ciclo saludable

- Cada vuelta del bucle medio produce al menos una entrada en el Context Update Record
- El Iteration Brief puede articular aprendizajes concretos vinculados a señales específicas
- Los stakeholders de Fases 1-2 participan en el bucle externo
- El tiempo dedicado a captura manual disminuye progresivamente
- Los mismos problemas NO aparecen en ciclos consecutivos sin producir cambios

:::danger[Señales de ciclo enfermo]
- El bucle interno opera pero los bucles medio y externo no se ejecutan
- El Signal Log crece pero el Context Update Record no
- Las sesiones de interpretación producen discusiones pero no actualizaciones
- El Iteration Brief es un copy-paste del anterior con fechas cambiadas
:::
