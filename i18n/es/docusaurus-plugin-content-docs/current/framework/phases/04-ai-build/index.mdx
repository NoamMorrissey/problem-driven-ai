# Framework: AI Build

## Construir bien porque se pensó bien

Esta sección contiene la guía operativa para ejecutar la Fase 4. Mientras la [sección de metodología](/fases/ai-build) explica el porqué y los conceptos, aquí se detalla el **cómo**: los pasos concretos, las plantillas, los artefactos y los criterios de validación.

### El proceso en cinco pasos

| Paso | Nombre | Output principal |
|---|---|---|
| 1 | **Setup del entorno** | Equipo alineado, grafo verificado, Decision Log operativo |
| 2 | **Construcción por Story Files** | Story Files completados y validados por QA |
| 3 | **Gestión de gaps** | Gaps clasificados y resueltos, Reglas actualizadas |
| 4 | **Revisión de fidelidad** | Build Validation Report firmado |
| 5 | **Cierre del Decision Log** | Decision Log consolidado, Iteration Brief |

### Artefactos de esta fase

- **Decision Log distribuido** — Un log por track con clasificación de impacto cruzado
- **Grafo de dependencias actualizado** — Documento de referencia para activar/desactivar tracks
- **Actas de sincronización de integración** — Contratos de interfaz y resoluciones
- **Build Validation Report** — Firmado por Tech Lead y stakeholders de Fases 1-2
- **Context Document actualizado** — Reglas emergidas de la construcción incorporadas
- **Iteration Brief (borrador)** — Aprendizajes abiertos para la Fase 5
