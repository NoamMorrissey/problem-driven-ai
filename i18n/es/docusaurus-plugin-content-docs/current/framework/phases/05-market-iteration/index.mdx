# Framework: Market Iteration

## El mercado como profesor permanente

Esta sección contiene la guía operativa para ejecutar la Fase 5. Mientras la [sección de metodología](/es/fases/market-iteration) explica el porqué y los conceptos, aquí se detalla el **cómo**: los pasos concretos, las plantillas, los artefactos y los criterios de validación.

### El proceso en cinco pasos

| Paso | Nombre | Output principal |
|---|---|---|
| 1 | **Activación del sistema de medición** | KPIs conectados, Signal Log operativo, dashboards activos |
| 2 | **Captura de señales** | Signal Log alimentado continuamente por 3 capas |
| 3 | **Interpretación de señales** | Señales clasificadas con hipótesis de causa validadas |
| 4 | **Actualización del Context Document** | Context Update Record con trazabilidad completa |
| 5 | **Iteration Brief** | Puente documentado entre ciclos de aprendizaje |

### Artefactos de esta fase

- **Signal Log:** Registro estructurado de señales del mercado con fuente, hipótesis y urgencia
- **Context Update Record:** Trazabilidad entre señales del mercado y cambios del contexto
- **Iteration Brief:** Aprendizajes del ciclo, estado del Context Document, foco del próximo ciclo
- **KPI Dashboard:** Visualización en tiempo real del movimiento de KPIs con alertas
- **Context Document (versión viva):** project-context.md, PRD, Story Files y KPI Register actualizados
