# 4. AI build phase

## Construir bien porque se pensó bien

> *"El valor del Context Phase se mide en la Fase 4. Si el contexto fue preciso, la construcción es casi aburrida. Eso no es un problema. Es el objetivo."*

---

**Qué es:** El proceso de materializar el sistema de Agentes, Reglas y Skills en una solución funcional. No es creación, es traducción del contexto a código.

**Quién participa:** Context Engineer (propietario del contexto), Tech Lead (coherencia técnica), Dev Leads de track (ejecución), desarrolladores con agentes de IA (construcción), QA Agent (validación).

**Qué preguntas responde:** ¿El output es fiel al Problem Statement? ¿Los tracks paralelos producen un sistema coherente? ¿Los gaps descubiertos durante la construcción se gestionan sin introducir Context Debt?

**Qué produce:** Solución construida y validada, Decision Log consolidado, Build Validation Report firmado por stakeholders, Context Document actualizado, Iteration Brief para la Fase 5.

**Cuándo está completa:** Los stakeholders de las Fases 1 y 2 reconocen el sistema completo como expresión fiel del Problem Statement y el Solution Brief.
